# AI环保应急预案助手产品说明

## 1. 产品定位

AI环保应急预案助手，面向环保咨询公司、第三方服务机构以及企业环保岗位人员，提供一套“资料整理 + 缺项补问 + 预案生成 + 附件生成 + 校核 + Word导出”的完整流程。

它不是单纯的 AI 写作工具，而是一个围绕“提交专家评审前送审稿”打造的交付助手。产品目标是尽量减少返工，让经验丰富的从业人员提高交付效率，也让初学者在引导下做出一份可以提交专家评审的材料。

## 2. 解决的问题

传统应急预案编制，常见痛点主要有：

- 资料来源分散，企业基础信息、风险物质、应急资源、联系人等内容需要人工反复整理。
- 初稿撰写耗时长，章节容易遗漏，附件容易缺项。
- 初学者不知道先收集什么资料，也不知道缺什么内容。
- 不同项目正文、备案表、目录、发布令、编制说明之间容易出现信息不一致。
- 专家评审前往往需要多轮修改，前期初稿质量越差，返工越多。

本产品的核心价值，就是把这些工作前移到系统里，用规则和流程先把明显缺项、结构问题和基础一致性问题处理掉。

## 3. 当前可演示能力

当前内部 MVP 已具备以下能力：

- 项目创建与基础资料录入
- 上传参考资料并自动抽取文本
- 对抽取结果进行人工确认和局部编辑
- 自动提取关键信息并识别缺项
- 通过补问方式回填缺失字段
- 生成预案正文、备案表、备案目录、发布令、编制说明
- 生成附件模块，包括应急通讯录、应急物资清单
- 支持用户在生成后继续编辑附件表格
- 提供整稿预览和基础校核结果
- 导出 Word 送审稿

这意味着当前版本已经能够跑通一条完整链路：

上传资料 -> 确认抽取结果 -> 补充缺项 -> 生成送审稿 -> 调整附件 -> 导出 Word

## 4. 交付边界

当前版本的目标不是直接生成最终备案定稿，而是生成一份可以提交专家评审的送审稿。

正式交付逻辑应理解为：

1. 系统先生成结构较完整、内容较规范的送审稿。
2. 用户提交专家评审。
3. 根据专家意见进行人工修改或二次生成。
4. 最终形成正式交付稿。

因此，这个产品在首阶段承担的是“把初稿质量尽量做高”，而不是“完全替代专家评审和人工定稿”。

## 5. 适合的目标客户

首批最适合的客户类型：

- 环保咨询公司
- 第三方环保服务机构
- 承接应急预案编制业务的小团队

可兼容但不是首版主优化对象的用户：

- 企业内部环保负责人
- 刚接触应急预案编制的初学者

不过从产品设计上，当前版本已经开始兼顾“小白可用”，比如：

- 先抽取再确认，避免资料直接误用
- 缺项自动补问，避免不知道下一步做什么
- 附件表格可视化编辑，降低对 Word 深度排版能力的依赖

## 6. 收费建议

如果后续做成收费产品，建议采用“基础订阅 + 按项目收费”模式。

推荐的收费结构：

- 基础版：允许项目建档、资料上传、有限次数试生成
- 专业版：开放完整生成、附件编辑、Word导出
- 项目包：按单个项目或单次正式导出收费
- 增值服务：人工复核、专家校审、专属模板、私有化部署

这样做的好处是：

- 容易先让客户试用看到效果
- 付费点与“正式交付价值”直接绑定
- 便于后续叠加人工服务和高客单价版本

## 7. 首阶段不建议过早纳入的功能

为了保证产品质量，以下能力建议放到第二阶段：

- 自动登录政府或环保网站
- 自动填报外部系统
- 多地区深度规则适配
- 复杂多人协同审核流
- 完整移动端深度编辑

原因很简单：首阶段最重要的是“生成质量”和“减少返工”，不是功能越多越好。

## 8. 推荐推进路径

建议按下面四步推进：

1. 继续完善内部 MVP，把生成质量、附件一致性、送审稿导出打磨稳定。
2. 用真实项目试跑，沉淀字段字典、附件模板、常见补问和常见专家意见。
3. 在内部验证效果后，再封装成对外收费版本。
4. 待送审稿生成能力稳定后，再单独建设“自动填报模块”。

## 9. 当前阶段可对外怎么说

如果你现在要对外介绍这个产品，可以用一句比较稳妥的话：

“这是一个面向环保应急预案编制场景的 AI 送审稿生成助手，可以把资料抽取、缺项补问、正文生成、附件整理和 Word 导出串成一条流程，帮助机构更快产出可提交专家评审的预案材料，并尽量减少前期返工。”
