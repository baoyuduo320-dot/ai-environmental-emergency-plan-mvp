# AI环保应急预案助手公网部署说明

## 1. 目标

本说明用于把当前项目整理成“注册账号后即可发布”的状态。

推荐部署结构如下：

- 前端：部署到 Vercel
- 后端：部署到 Render

这样做的原因是：

- 前端是 Next 网页，适合直接放到 Vercel
- 后端是 FastAPI 服务，适合直接放到 Render
- 前后端分开后，后续更容易维护和替换

## 2. 当前仓库已准备好的内容

当前项目已经整理为以下部署方式：

### 前端

- 目录：`apps/web`
- 框架：Next.js 15
- 对外访问域名：由 Vercel 提供
- 前端调用方式：通过项目内 `/api/projects/...` 路由转发到后端

### 后端

- 目录：`apps/worker`
- 框架：FastAPI
- 健康检查地址：`/health`
- 已补充 `requirements.txt`
- 已补充 `render.yaml`

## 3. 你需要注册的账号

正式上线前，只需要准备两个账号：

1. Vercel
2. Render

建议都使用同一个 GitHub 账号登录，这样后面连接仓库最省事。

## 4. 推荐发布顺序

正确顺序是：

1. 先把代码上传到 GitHub 仓库
2. 先在 Render 发布后端
3. 拿到 Render 的公网地址
4. 再到 Vercel 发布前端
5. 在 Vercel 中配置 `WORKER_BASE_URL`
6. 重新部署前端

## 5. GitHub 准备

部署平台都需要从 GitHub 拉取代码，所以这一步是必需的。

建议仓库名可以直接使用：

`ai-environmental-emergency-plan-mvp`

## 6. Render 部署后端

### 方式一：使用 Blueprint

仓库根目录已经提供了 `render.yaml`。

在 Render 中连接 GitHub 仓库后，可以直接选择基于仓库创建服务。Render 会识别：

- 服务目录：`apps/worker`
- 安装命令：`pip install -r requirements.txt`
- 启动命令：`uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- 健康检查：`/health`

### 方式二：手动创建 Web Service

如果你不走 Blueprint，就手动填写以下内容：

- Root Directory：`apps/worker`
- Build Command：`pip install -r requirements.txt`
- Start Command：`uvicorn app.main:app --host 0.0.0.0 --port $PORT`

部署成功后，你会得到一个类似下面的地址：

`https://ai-emergency-plan-worker.onrender.com`

健康检查地址示例：

`https://ai-emergency-plan-worker.onrender.com/health`

## 7. Vercel 部署前端

在 Vercel 里导入 GitHub 仓库后，按下面方式设置：

- Framework Preset：Next.js
- Root Directory：`apps/web`

环境变量增加：

- `WORKER_BASE_URL`

变量值填写你刚才在 Render 得到的后端公网地址，例如：

`https://ai-emergency-plan-worker.onrender.com`

配置完成后，重新部署前端。

部署成功后，你会得到一个类似下面的公网地址：

`https://ai-environmental-emergency-plan-mvp.vercel.app`

## 8. 上线后如何检查是否正常

建议按这个顺序检查：

1. 打开 Render 的 `/health`，确认返回正常
2. 打开 Vercel 首页，确认能进入创建项目页
3. 创建项目
4. 上传一份资料
5. 确认可以抽取文本
6. 确认可以生成内容
7. 确认可以导出 Word

如果前端能打开，但生成失败，优先检查：

- Vercel 是否配置了 `WORKER_BASE_URL`
- `WORKER_BASE_URL` 是否写成完整的 `https://...`
- Render 服务是否正在运行

## 9. 当前阶段的现实建议

如果你现在还没有 GitHub、Vercel、Render 账号，最省事的实际顺序是：

1. 先注册 GitHub
2. 再注册 Render
3. 先发后端
4. 再注册 Vercel
5. 最后发前端
